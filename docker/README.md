# Инфраструктура брокера (Kafka / MQTT)

## Учётные записи Kafka

Пользователи Kafka задаются в **переменных окружения** при запуске контейнеров. Файл **`docker/.env`** (скопируйте из `example.env`) используется и для старта Kafka, и для сервисов (компоненты, системы).

### Где прописывать

1. **`docker/.env`** (или тот `.env`, который передаётся в `docker compose --env-file`) — здесь задаются логины и пароли.
2. При старте контейнера **Kafka** эти переменные читаются и в JAAS добавляются соответствующие пользователи.
3. В **docker-compose** своих приложений (компонентов/систем) в `environment` задаёте `BROKER_USER` и `BROKER_PASSWORD` — те же самые логин/пароль, которые есть в `.env` и заведены в Kafka.

### Переменные в `docker/example.env`

| Переменная | Описание |
|------------|----------|
| `ADMIN_USER`, `ADMIN_PASSWORD` | Пользователь по умолчанию, всегда добавляется в Kafka. Подходит для тестов и сервисов без отдельных кредов. |
| `COMPONENT_USER_A`, `COMPONENT_PASSWORD_A` | Опционально. Если заданы — добавляются в Kafka; контейнер может подключаться под этим пользователем (`BROKER_USER=${COMPONENT_USER_A}`). |
| `COMPONENT_USER_B`, `COMPONENT_PASSWORD_B` | Аналогично для второго компонента. |

### Как добавить своих пользователей

- **Вариант 1:** Задать в `.env` пары `COMPONENT_USER_*` / `COMPONENT_PASSWORD_*` (сейчас поддерживаются A и B). При старте Kafka они автоматически попадут в JAAS.
- **Вариант 2:** Использовать одного `ADMIN_USER` для всех сервисов — в compose задать `BROKER_USER=${ADMIN_USER}`, `BROKER_PASSWORD=${ADMIN_PASSWORD}`.
- **Вариант 3:** Добавить в корневой `docker/docker-compose.yml` в сервис `kafka` новые переменные (например, `MY_USER`, `MY_PASSWORD`) и в `command` дописать строку в JAAS по аналогии с `COMPONENT_USER_A`/`COMPONENT_PASSWORD_A`.

Главное: **логин/пароль в Kafka (в JAAS) и в контейнере приложения (BROKER_USER/BROKER_PASSWORD) должны совпадать** и задаваться в одном и том же `.env`.

При запуске из `dummy_component` или `dummy_system` передаётся один `--env-file` (например, `dummy_component/docker/.env`); этот же файл используется для подстановки переменных в конфиг Kafka, так что пользователей достаточно прописать один раз в этом `.env`.
