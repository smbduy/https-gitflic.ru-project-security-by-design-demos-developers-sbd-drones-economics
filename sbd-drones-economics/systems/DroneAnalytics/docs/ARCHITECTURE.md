## Общие принципы
 - изоляция
 - контроль взаимодействия
 - минимализация доверенной вычислительной базы
 - использование брокера сообщений, мониторов безропасности
 - Контейнеризация, IaC
## Базовые принципы (интерпретация в контексте настоящей компоненты)
 - Взаимодействие между доменами безопасности происходит только через брокер сообщений
 - Выдача событий безопасности по запросу.
 - Передача телеметрик в сервис аналитики
 - Передача событий безопасностей в сервис аналитики
 - События должны записываться в журнал.
 - Сертификаты будут выдаваться и подписываться Регулятором. (То есть не сами подписываем TLS, а получаем его - Вроде)
## Основные функции
### Инфопанель
- Отображение наличия узлов в сети (см. диаграмму системы систем)
- Отображение списка выполняемых заказов с их свойствами (задача, цели безопасности, какой эксплуатант и дрон выполняет)
- Отображение состояния счетов основных игроков (Агрегатор, Эксплуатант, Страховая компания, Разработчик, Регулятор, ОрВД БАС, Заказчик)
- Отображение количества выполненных заказов
- Oтображение количества инцидентов с нарушением целей безопасности
### Журнал событий (K11)
 - Инициализация хранилища для записи событий 
 - Запись нового события
 - Выгрузка записанных событий
# Архитектура (GigaCode generated)

Проект DroneAnalytics представляет собой систему аналитики и информационных панелей, построенную на микросервисной архитектуре с использованием Docker-контейнеров. Основные компоненты системы:

1. Backend - сервис на Python с использованием FastAPI, отвечающий за бизнес-логику, аутентификацию и API для работы с логами. Обрабатывает три типа логов: телеметрию, базовые события и события безопасности. Использует Elasticsearch для хранения данных.
2. Frontend - веб-интерфейс на TypeScript/React с использованием Vite, предоставляющий панели для просмотра различных типов логов (события, безопасность, телеметрия) и аутентификации. Использует React Router для навигации между страницами.
3. Elastic - Elasticsearch 9.3.1 для хранения, поиска и анализа логов. Данные разнесены по индексам: telemetry, basic, event, safety.
4. Init-elastic - сервис инициализации Elasticsearch, который создает необходимые индексы с определенными маппингами при старте системы. Использует Python с библиотеками requests и pyyaml.
5. Proxy - Nginx как обратный прокси для маршрутизации запросов между клиентом и микросервисами, обеспечивая единый точку входа.

Сервисы взаимодействуют через Docker-сеть DA-net. Запуск и оркестрация осуществляются через docker-compose. Frontend и Backend зависят от успешного запуска init-elastic, который в свою очередь зависит от готовности Elastic. Система использует secrets для хранения чувствительных данных (сертификаты, конфигурация) и configs для конфигурационных файлов. Аутентификация реализована через JWT-токены с механизмом refresh.
