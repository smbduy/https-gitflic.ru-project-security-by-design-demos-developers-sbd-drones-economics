# Телеметрия: доверенные отправители и топики

## Кто может запрашивать get_state у телеметрии

Компонент **telemetry** обрабатывает запрос `get_state` **только от доверенного отправителя**:  
`sender` должен начинаться с `"security_monitor"` (т.е. запрос должен приходить через монитор безопасности).

По политикам МБ запрос в топик телеметрии с действием `get_state` разрешён только для отправителя **limiter**:

- **limiter** → МБ → топик `agrodron.telemetry`, action `get_state`.

Итого: по текущим политикам **доверенный** в смысле «кто может вызвать get_state у телеметрии через МБ» — это **limiter**. Любой другой отправитель без соответствующей политики МБ отклонит запрос.

## Какие топики использует телеметрия

| Назначение | Топик | Действие |
|------------|--------|----------|
| Куда телеметрия **шлёт** запросы | `agrodron.security_monitor` | `proxy_request` (цели — motors и sprayer) |
| Цель запроса 1 | `agrodron.motors` | `get_state` |
| Цель запроса 2 | `agrodron.sprayer` | `get_state` |

То есть телеметрия **нигде не публикует** сама (кроме ответа на свой топик при get_state). Она только запрашивает через МБ состояние motors и sprayer и отдаёт собранный снимок по get_state от доверенного отправителя.

## Итог

- **Доверенные отправители для телеметрии (get_state):** по политикам — только **limiter** (через МБ).
- **Доверенные топики «для телеметрии»:** телеметрия сама ходит только в топик **security_monitor**; цели запросов по политикам — **motors** и **sprayer** (get_state).
